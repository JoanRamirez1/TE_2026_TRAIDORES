Ingeniería de Software

Fundamentos, principios y el ciclo de vida del desarrollo de software moderno.

¿Qué es la Ingeniería de Software?

La ingeniería de software es una disciplina que aplica principios de ingeniería al diseño, desarrollo, prueba, mantenimiento y gestión de sistemas de software. A diferencia de la programación aislada, abarca todo el ciclo de vida del producto: desde la captura de requisitos hasta el retiro del sistema.

Fue reconocida formalmente como disciplina en la Conferencia de la OTAN de 1968, donde se identificó la llamada "crisis del software": los proyectos se entregaban tarde, fuera de presupuesto y con defectos graves. Desde entonces, la disciplina ha evolucionado para ofrecer métodos sistemáticos que garanticen calidad y predictibilidad.

Principios Fundamentales

Abstracción

Identificar los aspectos esenciales de un sistema ignorando los detalles irrelevantes para reducir la complejidad.

Modularidad

Dividir el software en componentes independientes con responsabilidades bien definidas, facilitando el mantenimiento.

Encapsulamiento

Ocultar los detalles internos de un módulo y exponer solo la interfaz necesaria para interactuar con él.

Reutilización

Crear componentes genéricos que puedan usarse en múltiples contextos, reduciendo costo y tiempo de desarrollo.

Rigor y Formalidad

Aplicar métodos precisos y verificables para garantizar que el software se comporta según lo especificado.

Verificación y Validación

Distinguir entre "¿construimos el producto correctamente?" y "¿construimos el producto correcto?"

Ciclo de Vida del Software (SDLC)

El ciclo de vida del software es el conjunto de fases que atraviesa un sistema desde su concepción hasta su retiro. Define quién hace qué, cuándo y cómo a lo largo del proyecto.

01

Planificación

Se define el alcance, se estiman recursos, tiempos y costos. Se evalúa la viabilidad técnica y económica del proyecto.

02

Análisis de Requisitos

Se recaban y documentan las necesidades del cliente. Se distingue entre requisitos funcionales y no funcionales.

03

Diseño

Se define la arquitectura del sistema: cómo se estructuran los módulos, qué tecnologías se usarán y cómo se almacenarán los datos.

04

Implementación

Los desarrolladores escriben el código fuente siguiendo los estándares y el diseño acordado.

05

Pruebas

Se verifica y valida el software mediante pruebas unitarias, de integración y de aceptación para detectar defectos antes de la entrega.

06

Despliegue y Mantenimiento

El software se instala en producción. El mantenimiento cubre corrección de errores, mejoras y adaptación a nuevos requisitos.

Modelos de Proceso

Cascada

Fases secuenciales y lineales. Simple de gestionar, pero rígido ante cambios de requisitos tardíos.

Espiral

Combina cascada con prototipado iterativo, añadiendo análisis de riesgos en cada ciclo.

Ágil

Entregas incrementales en sprints cortos. Alta adaptabilidad y colaboración continua con el cliente.

Incremental

Se construye el sistema por partes funcionales que se integran progresivamente hasta completar el producto.
