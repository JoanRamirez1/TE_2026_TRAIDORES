Patrones de Diseño

Un patrón de diseño es una solución general y reutilizable a un problema común que ocurre en el diseño de software. No es código terminado, sino una plantilla conceptual que te guía para estructurar el código de manera eficiente, mantenible y escalable.

Categorías Principales

Creacionales

Se centran en los mecanismos de creación de objetos, tratando de crear objetos de una manera adecuada a la situación. Ayudan a que el sistema sea independiente de cómo se crean, componen y representan sus objetos.

Estructurales

Se refieren a cómo se combinan clases y objetos para formar estructuras más grandes. Se aseguran de que, si una parte del sistema cambia, no sea necesario cambiar todo el sistema.

De Comportamiento

Se encargan de la comunicación entre objetos, cómo interactúan y cómo se reparten las responsabilidades. Ayudan a definir el flujo de control en sistemas complejos.

Ejemplos Clásicos

01
Singleton (Creacional)

Garantiza que una clase tenga una única instancia y proporciona un punto de acceso global a ella. Es muy utilizado para manejar conexiones a bases de datos o archivos de configuración.

02
Observer (Comportamiento)

Define una dependencia uno-a-muchos entre objetos, de forma que cuando un objeto cambia su estado, todos sus dependientes son notificados y actualizados automáticamente.

03
MVC (Arquitectónico)

Separa la lógica de negocio (Modelo), la interfaz de usuario (Vista) y la entrada del usuario (Controlador), permitiendo el desarrollo independiente y la reutilización de componentes.

Nota: Estos patrones son fundamentales para cumplir con los estándares de Ingeniería de Software solicitados en el proyecto.
